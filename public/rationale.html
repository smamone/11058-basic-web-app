I created a database that would track the DVDs users own.

I incorporated the following frameworks into my project:

  • Bulma
    Assisted with styling and consistency amongst fonts, responsiveness and spacing.
  • Font Awesome
    Enhanced the visual aspect of the site and improved communication with users through icons for buttons.

As I progressed through the project, I began to understand the purpose of php code more and how to apply it for various purposes, which may be due to the fact that I noticed similarities between php and JavaScript/jQuery.

As a result, I was able to implement several additional features to enhance the user’s overall experience on the site.

  • Scroll to top button
    Easy navigation to the top of the page when several records are displaying.
  • Record counter
  • When the user chooses “View all”, the total number of DVDs in the user’s collection based on the number of records within the database (not based on the last ID number, as records can be deleted) is displayed, giving the user a simple overview of the entire database.
  • When the user engages with the search field, the counter adjusts based on the number of DVDs currently displaying based on the search results. In future, I would also break down the total number by, for example, tv series and DVDs, and/or genre.
  • Sort options
    Allows sorting results by title, director, genre, tv and release date.
  • Confirmation messages/alerts
    To communicate to the user that their actions were successful or warn them prior to completing an irreversible action, such as:
  • Registration confirmation
  • Password reset
  • Adding a new record
  • Updating an existing record
  • Confirm leaving the edit page
  • Confirm deletion of a record
  • Adding images
    To provide the user with a greater visual concept of the record they are viewing, particularly with records with similar/identical titles and easily determine their differences. I achieved this by following code from the W3Schools website and Riley, and adjusting and applying it for my own requirements. I further built on this by adding an alt tag to each record’s image dependant on the DVD type, and to cater for empty image upload fields to ensure an error message did not display for the incorrect file type. The image process was a challenge to implement, particularly replacing an existing image attached to a record as it was more complex, but with some assistance I succeeded.

In allowing the user to specify whether a new DVD was a film or tv series, I incorporated a checkbox to check if it was a tv series. However, there were multiple issues in the checked value displaying as “Yes”, as well as ensuring the checkbox remained checked when the user entered the edit screen. I was able to implement if statements that read the value of each record to resolve both issues; one that displayed a tv tag on the read page if it was checked by the user in creation, and another that displayed a checked box when the user entered the edit screen.

To minimise confusion when displaying data, fields left blank in creation are hidden on the read page, such as the tv and season fields if the DVD is not part of a tv series. I achieved this by checking if the tv value is not NULL, and for all other fields checking if the field does not contain an empty string.

Although there were many challenges that arose during this project, the result was a database that met the minimum requirements and provided additional features to enhance the experience of the user.

In future, I would seek to implement a filtering function for results, a way of removing an existing image from a record, additional layout views (i.e. list), colour coding by genres and types (i.e. film, tv series, etc.), and a method of changing the input text colour when a change has been made on the edit page.
